## Полный контекст проекта

### Что это за проект
`rag_app` — production RAG-платформа (FastAPI + Qdrant + Qwen3-30B-A3B), которая индексирует
посты из Telegram-каналов и отвечает на вопросы пользователей с цитатами из первоисточников.

Это **портфолио-проект** для позиции Applied LLM Engineer. Проект должен демонстрировать
на интервью умение строить end-to-end RAG pipeline от ingestion до generation, с evaluation
и количественными метриками.

### Как коллекция будет использоваться

1. **Ingestion**: посты за 6+ месяцев скачиваются, разбиваются на чанки (smart chunking),
   эмбеддятся через Qwen3-Embedding-0.6B, индексируются в Qdrant (dense + sparse vectors)

2. **Retrieval**: hybrid search (RRF fusion dense+sparse → MMR-like re-scoring) +
   cross-encoder reranking (BGE-M3)

3. **Agent**: ReAct-подобный агент с native function calling
   (query_plan → search → rerank → compose_context → final_answer),
   coverage-based auto-refinement, citation-forced generation

4. **Evaluation**: по этой коллекции будет построен evaluation dataset из 50+ вопросов
   5 типов. Коллекция должна поддерживать ВСЕ типы:

   | Тип запроса | Пример | Что нужно от коллекции |
   |------------|--------|----------------------|
   | Фактический | "Когда вышла модель Qwen3?" | Каналы с точными датами, релизами, анонсами |
   | Аналитический | "Какие тренды в open-source LLM в 2026?" | Каналы с авторской аналитикой, обзорами |
   | Temporal | "Что нового за последний месяц в NLP?" | Плотный поток постов с датами |
   | Сравнительный | "GPT-4o vs Claude 3.5 для кодинга" | Каналы с бенчмарками, сравнениями моделей |
   | Multi-hop | "Как выход Llama 3 повлиял на рынок?" | Одно событие в разных каналах, разные мнения |

5. **Benchmark**: коллекция будет использоваться для сравнения нашего pipeline
   с LlamaIndex/LangChain baseline. Нужна достаточная сложность и разнообразие
   чтобы показать преимущество adaptive retrieval.

### Техническая специфика
- Язык постов: русский (основной). Каналы на английском не подходят.
- Формат: Telegram посты (текст, иногда с картинками/ссылками).
  Видео/аудио контент не индексируется.
- Объём: нужно минимум 5000–10000 точек (чанков) после chunking.
  При среднем 1 пост = 1.3 чанка, нужно ~4000–8000 постов суммарно за полгода.
- Smart chunking: посты <1500 символов идут целиком, >1500 разбиваются рекурсивно.

## Критерии отбора каналов

### ОБЯЗАТЕЛЬНЫЕ (канал не включается без соответствия ВСЕМ):

1. **Авторитетность и экспертиза автора**
   - Автор(ы) идентифицируемы (не анонимный канал)
   - У автора есть credentials: работа в ML/AI компании, исследовательская деятельность,
     преподавание, публикации, опыт в индустрии
   - Или: канал ведётся командой/организацией с признанной экспертизой
   - Evidence: ссылка на профиль автора, место работы, публикации

2. **Активность и стабильность**
   - Минимум 3 поста в неделю в среднем
   - Канал активен в 2025–2026 (не заброшен)
   - Стабильная активность (не "10 постов в один день, потом тишина месяц")

3. **Оригинальный контент**
   - Авторские тексты: разборы, анализ, мнения, обзоры
   - НЕ чистый агрегатор/репостер чужого контента
   - Допустимо: авторский комментарий к новостям, переводы с добавленным анализом

4. **Фактологическая плотность**
   - Посты содержат проверяемые факты: даты релизов, названия моделей/фреймворков,
     метрики/бенчмарки, ссылки на papers/GitHub/блоги
   - Не "мотивационный контент", не "10 способов заработать на AI",
     не промо-контент курсов

5. **Русский язык**
   - Основной язык канала — русский
   - Отдельные посты на английском допустимы, но bulk контента на русском

### ЖЕЛАТЕЛЬНЫЕ (усиливают позицию канала):

- Наличие технических деталей (архитектуры, код, конфиги)
- Обсуждение trade-offs и ограничений (не только хайп)
- Ссылки на первоисточники (papers, GitHub, блоги компаний)
- Высокая плотность AI/ML контента (не "обо всём понемногу")
- Размер аудитории 5K+ подписчиков (социальное доказательство качества)

## Необходимое покрытие тематик

Коллекция должна покрывать ВСЕ следующие области. Для каждой области нужно
минимум 2 канала (для cross-reference и multi-hop запросов):

| # | Область | Зачем для RAG evaluation | Мин. каналов |
|---|---------|--------------------------|--------------|
| 1 | LLM новости и релизы | Фактические запросы, temporal queries | 3+ |
| 2 | Научные статьи / research papers | Аналитические запросы, deep technical | 2+ |
| 3 | Applied ML / production | Инженерные запросы, best practices | 2+ |
| 4 | Open-source модели и инструменты | Сравнительные запросы, бенчмарки | 2+ |
| 5 | AI индустрия, стартапы, продукты | Multi-hop (связь событий и рынка) | 2+ |
| 6 | NLP / обработка текста | Узкоспециализированные запросы | 1+ |
| 7 | MLOps / инфраструктура / deployment | Инженерная зрелость | 1+ |
| 8 | Computer Vision | Расширение domain coverage | 1+ |
| 9 | Data Science / аналитика | Широкая аудитория, базовые запросы | 1+ |
| 10 | AI этика, регулирование, безопасность | Нетривиальные аналитические запросы | 1+ |

## Итоговая сводка

После списка каналов предоставь:

1. **Coverage matrix**: таблица "канал × тематическая область" (✅/❌)
2. **Оценка объёма**: примерный расчёт количества постов за 6 месяцев по всем каналам
3. **Gaps**: какие тематики покрыты слабо и что можно добавить
4. **Рекомендации по приоритету**: если нужно выбрать 15–20 из предложенных,
   какие наиболее ценны и почему
5. **Рекомендации по временному окну**: оптимальный период для ingestion
   (чтобы поймать ключевые события AI/ML мира)
