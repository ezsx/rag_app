# final_answer

`src/services/tools/final_answer.py` нормализует финальный payload агента и проверяет качество citations.

## Что делает

- извлекает ссылки `[N]` прямо из текста ответа
- объединяет их с `sources`, пришедшими от LLM
- если источники не указаны вообще, добавляет warning в текст ответа
- логирует количество предложений без citations для observability

## Правила

### Merge sources

Если LLM вернул:

- `answer = "DeepSeek выпустит V4 [2]"`
- `sources = [1]`

то итоговый payload будет:

```python
{
    "answer": "...",
    "sources": [1, 2],
}
```

### Warning при пустых citations

Если в ответе нет ни `sources`, ни `[N]` внутри текста, tool дописывает:

```text
⚠️ Источники не указаны. Информация может быть неточной.
```

### Observability

Tool не удаляет предложения без citations, но пишет лог:

```text
final_answer: X/Y предложений без цитат
```

Это нужно для мониторинга галлюцинаций без агрессивной постобработки ответа.
