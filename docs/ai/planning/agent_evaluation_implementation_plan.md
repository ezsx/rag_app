# План реализации инструмента оценки ReAct агента

## Подход к использованию исследования

### Шаг 1: Извлечение ключевых решений

Из исследования нужно извлечь и зафиксировать:

1. **Выбранные метрики** (что измеряем):
   - Список метрик с приоритетами (MVP vs расширенная версия)
   - Формулы и алгоритмы расчета
   - Автоматизация vs ручная проверка

2. **Архитектура решения**:
   - Структура скрипта/эндпоинта
   - Формат датасета
   - Формат отчета

3. **Технические детали**:
   - Как вызывать Agent API
   - Как вызывать Baseline QA API
   - Как извлекать метрики из ответов
   - Как сравнивать результаты

### Шаг 2: Создание технической спецификации

На основе исследования создать упрощенную спецификацию для реализации:

- **Входные данные**: формат датасета
- **Процесс**: пошаговый алгоритм оценки
- **Выходные данные**: формат отчета
- **Метрики**: детали расчета каждой метрики

### Шаг 3: Поэтапная реализация

Начинать с MVP (минимально жизнеспособного продукта), затем расширять.

---

## Этап 1: MVP - Базовая структура и простые метрики

### Цель: Запустить базовую оценку агента на датасете

**Что реализуем**:
1. ✅ Загрузка датасета из `datasets/eval_questions.json`
2. ✅ Вызов Agent API для каждого вопроса
3. ✅ Вызов Baseline QA API для каждого вопроса
4. ✅ Извлечение базовых метрик из ответов:
   - Citation Coverage (уже есть в ответе агента)
   - Latency (время выполнения)
   - Refinement Count (количество refinement раундов)
   - Verification Confidence (если есть)
5. ✅ Сохранение результатов в JSON файл
6. ✅ Простой текстовый отчет (сравнение Agent vs Baseline)

**Что НЕ делаем на этом этапе**:
- ❌ Сложные метрики (Citation Precision, Recall@5)
- ❌ Автоматическая оценка качества ответов
- ❌ Визуализации и графики
- ❌ Детальная диагностика ошибок

**Результат**: Работающий скрипт, который может запустить оценку и показать базовые метрики.

---

## Этап 2: Расширение метрик

### Цель: Добавить автоматические метрики качества

**Что добавляем**:
1. ✅ Recall@5 (был ли нужный документ в топ-5 результатов поиска)
2. ✅ Citation Precision (автоматическая проверка через LLM-as-judge или эвристики)
3. ✅ Answer Length (длина ответа для оценки Conciseness)
4. ✅ Улучшенный отчет с детализацией по метрикам

**Результат**: Более полная оценка качества ответов.

---

## Этап 3: Диагностика и анализ

### Цель: Понять причины ошибок

**Что добавляем**:
1. ✅ Анализ паттернов ошибок (по категориям вопросов)
2. ✅ Детальная диагностика неудачных ответов
3. ✅ Сравнение шагов агента (какие инструменты использовались)
4. ✅ Визуализации (графики метрик, сравнения)

**Результат**: Инструмент для диагностики и улучшения агента.

---

## Конкретный план действий для начала

### День 1: Структура и базовая функциональность

1. **Создать структуру скрипта** (`scripts/evaluate_agent.py`):
   ```python
   # Основные функции:
   - load_dataset() - загрузка eval_questions.json
   - call_agent_api(query) - вызов /v1/agent/stream
   - call_baseline_api(query) - вызов /v1/qa
   - extract_metrics(agent_response, baseline_response) - извлечение метрик
   - save_results(results) - сохранение в JSON
   - generate_report(results) - простой текстовый отчет
   ```

2. **Расширить датасет** (если нужно):
   - Добавить поля для метаданных (если исследование рекомендует)
   - Сохранить текущий формат, но подготовить к расширению

3. **Реализовать базовые вызовы API**:
   - Подключение к Agent API (SSE стриминг)
   - Подключение к Baseline QA API
   - Обработка ошибок и таймаутов

### День 2: Извлечение метрик

1. **Реализовать извлечение метрик из ответов**:
   - Citation Coverage (из события `final` агента)
   - Latency (время от начала до `final` события)
   - Refinement Count (из метаданных `final`)
   - Verification Confidence (если есть)

2. **Реализовать сохранение результатов**:
   - Структура JSON для результатов
   - Сохранение по каждому вопросу
   - Агрегированные метрики (средние, процентили)

3. **Простой отчет**:
   - Таблица сравнения Agent vs Baseline
   - Базовые статистики

### День 3: Тестирование и отладка

1. **Запустить на текущем датасете** (8 вопросов)
2. **Проверить корректность метрик**
3. **Исправить ошибки**
4. **Документировать использование**

## Рекомендации по работе с исследованием

### 1. Создай чеклист из исследования

Выпиши из исследования:
- [ ] Какие метрики выбраны для MVP
- [ ] Формулы расчета метрик
- [ ] Структура датасета
- [ ] Формат отчета
- [ ] Алгоритмы расчета (псевдокод)

### 2. Создай упрощенную спецификацию

На основе исследования создай документ `docs/ai/planning/agent_evaluation_spec.md`:
- Входные данные (формат датасета)
- Процесс оценки (пошагово)
- Метрики (формулы, алгоритмы)
- Выходные данные (формат отчета)

### 3. Начинай с простого

Не пытайся реализовать всё сразу:
- Сначала базовый скрипт, который просто вызывает API
- Потом добавь извлечение одной метрики
- Потом добавь следующую метрику
- И так далее

### 4. Тестируй на малом датасете

Начни с 2-3 вопросов из датасета:
- Проверь, что вызовы API работают
- Проверь, что метрики извлекаются правильно
- Затем расширь до полного датасета

### 5. Используй исследование как справочник

Не нужно помнить всё наизусть:
- Держи исследование открытым
- Ссылайся на него при реализации каждой метрики
- Используй как чеклист (что уже сделано, что осталось)
